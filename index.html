GhostVault Lab

Free up space on your computer without losing your files. They leave as encrypted pieces and come back exactly the same whenever you want. Free, secure cloud storage made from GitHub repos you have access to.

By ABProjects · alibaghi03@gmail.com

What it is

Free up your disk, store your files for free

GhostVault Lab is a small desktop app that frees up space on your computer. You point it at a file, it cuts that file into small pieces, encrypts each one, and spreads them across GitHub repos you have access to. The file then leaves your machine having been completely deleted. The room it was taking becomes free again and the device becomes lighter as memory gets freed.

Whenever you want it back, the app pulls the pieces in, checks them, decrypts them, and rebuilds the exact same file. The file is off your disk but never really gone. It returns the moment you ask for it, as if it never left. Putting a file away and pulling it back are both just a couple of clicks.

That is the second half of the idea. GitHub is normally for code, but the private repos you have are free space sitting on the web. GhostVault puts them to work as your own free, secure cloud storage. You clear room on your device and get cloud storage for nothing, both from the same move.

Nothing about your file is ever readable online, since every piece is locked before it leaves you, and nothing opens or runs on its own. It all runs on your machine and only talks to the repos you connect. It is still an experiment, so think of it as a way to free up space, not as the only home for something you cannot afford to lose.

Unsigned builds

These apps are not code-signed, so the first time you open one, Windows and macOS will warn you about an unknown developer. That is expected. The next section shows how to get past it.

Get started

Install and run

Windows

Download GhostVault-Windows.zip and unzip it. Run GhostVault.exe. If a blue SmartScreen warning appears, click More info and then Run anyway. The warning shows only because the app is unsigned.

macOS

Download the zip for your chip, Apple Silicon for M1 through M4 or Intel for older Macs, and unzip it. Open the .dmg inside and drag GhostVault to your Applications folder. The first time you launch it, right-click the app and choose Open, which lets macOS run an unsigned app.

Linux

Download GhostVault-Linux.zip and unzip it. In a terminal, make it runnable with chmod +x GhostVault, then start it with ./GhostVault. You need a desktop environment for the window to appear.

The idea

Why split the file up

The simplest approach would be to take your whole file and drop it in one place. That falls apart fast. A big file is awkward to keep in a code repository, and GitHub flatly refuses any single file over 100 MB. One large blob sitting in one repo is the kind of thing these services are not built to hold.

So GhostVault slices the file into small pieces, around half a megabyte each. Small pieces upload without trouble, they spread across as many repositories as you connect so no single repo carries the whole thing, and they download all at once when you restore, which keeps rebuilding quick.

Encryption is the other half of it. Those pieces land in storage you do not fully control. A repo can be shared, or made public by accident, or simply somewhere you would rather not leave a readable file. So every piece is locked before it ever leaves your computer. On its own, a piece is just scrambled noise that opens into nothing.

Put together, splitting handles the size and keeps the load spread out, and encryption keeps the pieces private. Neither half does much without the other.

How it works

Ghosting a file

When you add a file, the app does five things in order. Each piece is locked on its own before it ever leaves your machine, and the original is only removed once everything else has succeeded.

The encryption is per file. Every file gets its own random key, and every chunk gets its own random one-time value, so the pieces sitting in your caves are unreadable on their own.

The pieces

Where everything lives

Two parts work together. The encrypted chunks live out in your caves, spread across the repositories you connected. The manifest, which holds the key and the map of which chunk went where, stays on your computer and never leaves it.

This split is the whole idea. Anyone who can read your repositories only finds scrambled pieces. The thing that turns those pieces back into a file lives only with you.

How it works

Restoring a file

Restoring runs the process in reverse. The app pulls the pieces down together rather than one at a time, so even a file split into many pieces comes back quickly. It makes sure each piece is intact, and only writes the final file once it has confirmed the result matches the original exactly.

If any check fails along the way, nothing is deleted and nothing is overwritten, so a bad restore leaves your ghost untouched.

Setup

Connecting your caves

A cave is somewhere the app can store encrypted pieces. You add them in Settings, and you need at least one verified cave before you can ghost anything. The more you add, the more your pieces spread out, and the less any single repo has to carry.

GitHub

Use a GitHub repository you own. You give the app the repository URL, written like https://github.com/owner/repo, and a personal access token that can write files to it. A classic token needs the repo scope. A fine-grained token needs Contents read and write on that repository.

Two small things make a real difference. Use private repositories, since the pieces are encrypted either way but a private repo also keeps them out of public view. And connect as many repos as you can. The more there are, the more the pieces spread out, so no single repo fills up or runs into its limits. With this app, more is simply better.

Storage

Where your data is kept

GhostVault keeps its records on your own machine, not next to the app and not on the internet. It is stored per user, so each account has its own separate vault, and it survives app updates because it lives outside the app itself.

  • On Windows, in AppData\Roaming\GhostVaultLab under your user.
  • On macOS and Linux, in a hidden GhostVaultLab folder in your home directory.

Good to know

What is protected, and what to mind

What is protected

The pieces uploaded to your caves are encrypted before they leave your machine, so anyone who reads those repositories only sees scrambled data they cannot open.

What to keep safe

The key that unlocks everything lives only in the local manifest on your computer. There is no copy in the cloud and no backup. If you lose the app's data folder, the ghosted files cannot be recovered. Treat that folder as carefully as the files themselves.

A few more things

The original file is removed permanently after a successful ghost, so only ghost files you can afford to commit. A restored file is never opened or run for you. And because the builds are unsigned, your system will warn you the first time you open the app, which is normal.
